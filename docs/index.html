Key Features

What makes the Science Data Lake different from using individual sources.

🔗

Cross-Source Record Linkage

293M papers linked across 8 sources via DOI normalization. Compare citation counts, disruption scores, and retraction status in a single query.

🧬

13 Scientific Ontologies

1.3M terms from MeSH, GO, ChEBI, CSO, and 9 more — all linked to OpenAlex's 4,516 topics via embedding-based alignment (99.8% coverage).

💾

Views-over-Parquet

~960 GB of Parquet files, a 500 KB DuckDB file with 151 views. Fully portable: mount the drive, regenerate views, query. No ETL server needed.

🔄

Reproducible Pipeline

One CLI command downloads, converts, links, and materializes everything. Idempotent updates — refresh any source to its latest snapshot.

🤖

LLM-Ready Documentation

SCHEMA.md provides every table, column, type, join strategy, and performance tier. Point your AI coding agent at it and query in natural language.

🌍

Remote Access via HuggingFace

Query Parquet files directly from HuggingFace using DuckDB's hf:// protocol — no download required.

Architecture

A lightweight DuckDB database defines 151 SQL views pointing to Parquet files. No data is copied into the database — only view definitions.

datalake.duckdb (500 KB, view definitions only) ├── s2ag.* → datasets/s2ag/parquet/**/*.parquet (437 GB) ├── openalex.* → datasets/openalex/parquet/**/*.parquet (262 GB) ├── sciscinet.* → datasets/sciscinet/{core,large}/*.parquet (151 GB) ├── pwc.* → datasets/paperswithcode/parquet/*.parquet (6.2 GB) ├── retwatch.* → datasets/retractionwatch/parquet/*.parquet (70 MB) ├── ros.* → datasets/reliance_on_science/parquet/*.parquet (2.7 GB) ├── p2p.* → datasets/preprint_to_paper/parquet/*.parquet (735 MB) ├── {ont}.* → datasets/{ont}/parquet/*.parquet (56 MB) ├── xref.doi_map → UNION ALL across datasets (normalized DOIs) (588M rows) └── xref.unified_papers → Materialized cross-source join (293M rows)

Source Coverage & Overlap

The 293M-row unified_papers table links papers across all sources via DOI normalization. OpenAlex provides 99.7% coverage; SciSciNet and S2AG each cover ~50%.

Embedding-Based Ontology Alignment

OpenAlex's 4,516 topics are mapped to 13 scientific ontologies using BGE-large-en-v1.5 embeddings + FAISS nearest-neighbour search, covering 99.8% of topics with 16,150 mappings. This is a 17× improvement over string-matching baselines.

Quick Start

Query directly from HuggingFace — no download required.

import duckdb

con = duckdb.connect()
con.execute("INSTALL httpfs; LOAD httpfs;")

# Top 100 most disruptive papers
df = con.execute("""
    SELECT doi, title, year, sciscinet_disruption, oa_cited_by_count
    FROM 'hf://datasets/J0nasW/science-datalake/xref/unified_papers/*.parquet'
    WHERE sciscinet_disruption IS NOT NULL
    ORDER BY sciscinet_disruption DESC
    LIMIT 100
""").df()

Build the Full Instance Locally

# Clone and run the pipeline
git clone https://github.com/J0nasW/science-datalake
cd science-datalake
python scripts/datalake_cli.py download --all
python scripts/datalake_cli.py convert --all
python scripts/create_unified_db.py
python scripts/materialize_unified_papers.py

# Update any source to its latest snapshot
python scripts/datalake_cli.py update openalex
